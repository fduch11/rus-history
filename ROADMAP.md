# История Руси — дорожная карта автономной разработки

## Цель проекта

Создать полностью автономное мобильное одностраничное веб-приложение «История Руси», которое:

- работает локально при открытии `index.html`;
- публикуется через GitHub Pages;
- не использует npm, React, Vue, внешние библиотеки, CDN, API и `fetch`;
- адаптировано под iPhone;
- хранит пользовательские данные в `localStorage`;
- устанавливается как PWA при открытии через HTTPS;
- остаётся работоспособным после каждого коммита.

## Правила автономной разработки

1. Работать напрямую в ветке `main`.
2. Перед изменением читать этот файл и текущее состояние репозитория.
3. За одну итерацию выполнять один логически завершённый этап.
4. После каждой итерации:
   - проверять синтаксис JavaScript;
   - проверять локальные пути к CSS, JS, изображениям, manifest и service worker;
   - убеждаться, что приложение открывается без сервера;
   - делать отдельный коммит с конкретным сообщением;
   - обновлять статусы в этом файле.
5. Не запрашивать подтверждение пользователя между этапами.
6. Не удалять работающие функции без равноценной замены.
7. Не использовать внешние сетевые зависимости.
8. После завершения всех обязательных пунктов провести итоговый аудит, отметить проект как завершённый и больше не изменять репозиторий.
