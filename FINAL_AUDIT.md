# Итоговый аудит «История Руси»

Дата аудита: 17 июля 2026 года.

## Итог

Версия 2.0 завершена по дорожной карте. Репозиторий содержит автономное мобильное одностраничное приложение на HTML, CSS и чистом JavaScript, рассчитанное на прямое открытие `index.html` и публикацию через GitHub Pages.

## Проверено

### Структура и зависимости

- `index.html` подключает только локальные CSS, JavaScript, manifest, иконки и изображения.
- npm, React, Vue, внешние библиотеки, CDN и внешние API отсутствуют.
- В основном приложении отсутствуют вызовы `fetch()`.
- Service worker использует только Cache API и локальные ресурсы.

### Локальный запуск

- Все базовые модули подключаются относительными путями.
- Регистрация service worker выполняется только при HTTP/HTTPS.
- При `file://index.html` PWA-функции пропускаются, а лента, поиск, фильтры, карта, тесты и `localStorage` остаются доступны.

### PWA и офлайн-режим

- `manifest.webmanifest` содержит имя, язык, `start_url`, `scope`, цвета и две PNG-иконки.
- Service worker использует версионированный кэш `rus-history-v4`.
- Список предварительного кэширования содержит HTML, manifest, все подключённые CSS/JavaScript, иконки и локальные иллюстрации.
- Старые версии кэша удаляются при активации.
- Навигационные запросы имеют резервный ответ `index.html`.

### Исторические данные

- База содержит 108 событий.
- `validate-data.js` проверяет обязательные поля, типы, уникальность ID, хронологию, связи `related`, локальные изображения и безопасность путей.
- Валидатор не требует npm или внешних библиотек.

### Пользовательские функции

Статически проверено наличие модулей для:

- исторической ленты, поиска, фильтров и сортировки;
- подробного диалога и прямых ссылок;
- избранного, просмотренных и изученных событий;
- прогресса, экспорта, импорта и сброса данных;
- события дня и исторического теста;
- локальной карты;
- темы и трёх размеров текста;
- клавиатурной доступности;
- мобильной компоновки, safe-area, ориентации и экранной клавиатуры.

## Ограничение аудита

Из текущей рабочей среды не удалось выполнить полноценный интерактивный браузерный прогон опубликованного GitHub Pages и сценарий чистого профиля. Эти пункты проверены статически по подключениям, условиям регистрации service worker, структуре интерфейса и локальному хранению. Для независимой приёмки рекомендуется один ручной прогон по чек-листу ниже.

## Чек-лист ручной приёмки

1. Открыть `index.html` напрямую и проверить ленту, поиск, фильтры, модальное окно, тест и карту.
2. Открыть GitHub Pages в приватном окне и убедиться, что нет ошибок в консоли.
3. Перезагрузить страницу без сети после первого онлайн-запуска.
4. Проверить экспорт, сброс и импорт пользовательских данных.
5. Проверить iPhone SE и современный iPhone в портретной и альбомной ориентации.
6. Проверить навигацию клавишами Tab, Enter, Space и Escape.
