# История Руси

Автономное мобильное одностраничное веб-приложение с исторической лентой событий Руси и России.

Приложение написано на HTML, CSS и чистом JavaScript. Для основной работы не нужны npm, фреймворки, сервер, CDN, внешние библиотеки или API.

## Возможности

- 108 исторических событий, сгруппированных по векам;
- поиск по датам, событиям, местам, описаниям и личностям;
- фильтры по периоду, веку, избранному и непросмотренным событиям;
- сортировка по времени и названию;
- подробное окно события с причинами, ходом, последствиями, личностями, фактами, источниками и связанными событиями;
- прямые ссылки формата `#event-<id>`;
- избранное, просмотренные и изученные события;
- 6 маршрутов изучения с локальным прогрессом по траекториям;
- общий прогресс и прогресс по историческим периодам;
- событие дня;
- исторический тест пяти типов;
- локальная интерактивная SVG-карта;
- светлая и тёмная темы;
- три размера текста;
- экспорт, импорт и полный сброс пользовательских данных;
- PWA-установка и повторный запуск без сети;
- адаптация под iPhone, safe-area, экранную клавиатуру и смену ориентации;
- клавиатурная навигация и поддержка экранных дикторов.

## Локальный запуск без сервера

1. Скачайте или клонируйте репозиторий.
2. Откройте файл `index.html` в браузере.

Базовые функции работают при адресе вида `file:///.../index.html`.

В режиме `file://` браузер не регистрирует service worker, поэтому установка PWA и офлайн-кэш недоступны. Историческая лента, поиск, фильтры, тесты, карта и локальное хранение продолжают работать.

## Запуск через GitHub Pages

1. Откройте настройки репозитория GitHub.
2. Перейдите в раздел **Pages**.
3. Выберите публикацию из ветки `main` и корневой папки `/`.
4. Откройте выданный HTTPS-адрес.

При запуске по HTTPS регистрируется `service-worker.js`, после первого успешного открытия локальные ресурсы сохраняются для повторного запуска без сети.

## Установка как PWA

Через GitHub Pages или другой HTTPS-хостинг:

- в Safari на iPhone выберите **Поделиться → На экран «Домой»**;
- в Chrome или Edge используйте действие **Установить приложение**.

Файл `manifest.webmanifest` описывает приложение, а локальные иконки находятся в `icons/`.

## Структура проекта

- `index.html` — разметка интерфейса и подключение локальных модулей;
- `style.css` — базовые стили;
- `data.js` — основная историческая база;
- `event-details.js` — дополнительные подробные поля событий;
- `illustrations.js` — назначение локальных иллюстраций;
- `app.js` — состояние, хранение, фильтры, маршруты изучения и рендеринг ленты;
- `modal-details.js` — расширенное содержимое модального окна;
- `direct-links.js` — прямые ссылки на события;
- `navigation.js` — навигация по векам и возврат наверх;
- `progress.js` — прогресс, экспорт, импорт и сброс данных;
- `daily-event.js` — событие дня;
- `quiz.js` — исторический тест;
- `map.js` — локальная карта;
- `reading-settings.js` — размер текста и настройки чтения;
- `accessibility.js` — дополнительные функции доступности;
- `performance.js` — объединение частых событий интерфейса;

- `pwa.js` — регистрация service worker только по HTTP/HTTPS;
- `service-worker.js` — версионированный офлайн-кэш;
- `validate-data.js` — автономная Node-проверка исторической базы без браузера и DOM.

### Каталоги

- `images/` — локальные SVG-иллюстрации событий;
- `icons/` — PNG-иконки PWA.

## Структура исторического события

Каждое итоговое событие должно содержать поля:

```js
{
  id: 1,
  year: "862",
  sortYear: 862,
  century: "IX век",
  title: "Призвание Рюрика",
  period: "Древняя Русь",
  place: "Новгород",
  summary: "Краткое описание",
  description: "Полное описание",
  course: "Ход события",
  causes: ["Причина"],
  consequences: ["Последствие"],
  people: ["Историческая личность"],
  fact: "Интересный факт",
  sources: ["Источник"],
  related: [2, 3],
  image: "images/862-rurik.svg"
}
```

### Требования к полям

- `id` — уникальное положительное целое число;
- `year` — отображаемая дата или период;
- `sortYear` — числовое значение для хронологической сортировки;
- `century` — отображаемое название века;
- `related` — массив существующих ID без ссылки события на самого себя;
- `image` — относительный путь к существующему локальному изображению;
- текстовые поля не должны быть пустыми.

## Добавление события

1. Добавьте базовую запись в `data.js` в правильном хронологическом месте.
2. При необходимости дополните её в `event-details.js`.
3. Назначьте существующее изображение или добавьте новый локальный SVG в `images/`.
4. Проверьте ссылки `related`.
5. Запустите проверку данных:

```bash
node validate-data.js
```

Успешная структурная проверка завершается кодом `0`. При ошибках выводится список проблем и возвращается код `1`.
Скрипт рассчитан на запуск в Node и не требует браузерной среды.
Дополнительно валидатор выводит предупреждения, если поля событий выглядят шаблонными: например, если `related` сводится к соседним ID, а `description`, `course`, `fact`, `causes` и `sources` не дают содержательной детализации для конкретного события.

## Пользовательские данные

Основное состояние хранится в `localStorage` браузера.

Используются отдельные ключи:

- `rusHistory` — тема, фильтры, активный маршрут, избранное, просмотренные и изученные события;
- `rusHistoryQuiz` — лучший результат и статистика тестов;
- `rusHistoryReading` — размер текста и связанная настройка темы.

Данные остаются только в текущем браузере и не отправляются во внешние сервисы.

## Маршруты изучения

В приложении доступны 6 учебных маршрутов:

- становление государственности;
- борьба с внешними угрозами;
- реформы и переустройство;
- расширение пространства;
- революции и сломы эпох;
- наука, право и модернизация.

Выбор маршрута показывает компактную траекторию из связанных событий поверх общей базы. Прогресс маршрута вычисляется локально по уже изученным событиям и автоматически сохраняется вместе с основным состоянием приложения.

## Экспорт и восстановление

В панели прогресса доступны три действия:

- **Экспорт JSON** — сохраняет резервную копию прогресса, статистики тестов и настроек чтения;
- **Импорт JSON** — проверяет выбранный файл и восстанавливает все сохранённые пользовательские данные;
- **Сбросить данные** — удаляет прогресс, избранное, статистику тестов и настройки чтения после подтверждения.

Перед очисткой данных браузера рекомендуется выполнить экспорт JSON.

Экспорт использует формат резервной копии `rus-history-backup`. Импорт совместим и с этим полным форматом, и со старым JSON, который содержал только состояние `rusHistory`.

## Офлайн-кэш и обновления

`service-worker.js` предварительно кэширует HTML, CSS, JavaScript, manifest, иконки и иллюстрации.

Имя кэша содержит версию. При изменении списка ресурсов необходимо:

1. увеличить `CACHE_VERSION`;
2. добавить или удалить пути в `LOCAL_RESOURCES`;
3. убедиться, что каждый путь существует.

После активации новой версии старые кэши приложения удаляются.

Service worker не регистрируется в режиме `file://`, поэтому прямое локальное открытие не зависит от состояния кэша.

## Ограничения проекта

Не добавляйте в основное приложение:

- npm-зависимости;
- React, Vue и другие фреймворки;
- CDN;
- внешние API;
- сетевые изображения или шрифты;
- обязательный сервер;
- вызовы `fetch()` в основном приложении.

## Проверка перед изменениями

Перед публикацией изменений:

```bash
node --check app.js
node --check event-details.js
node --check illustrations.js
node --check progress.js
node --check validate-data.js
node validate-data.js
```

Также необходимо проверить локальные пути, запуск `index.html`, GitHub Pages, PWA и повторный офлайн-запуск.
